Yönlendirme Sistemi

Bu rehber, sıfırdan, çalışan bir prototip oluşturmak için ihtiyaç duyduğun birçok şeyi kapsamaktadır. Her bölümü sırasıyla oku ve uygulamalı çalış.

ESP32 + ESP-NOW 8×8 Dot Matrix + MT3608 BFS Algoritması Mesh Ağ
01

Sistem Mimarisi

Projeye başlamadan önce büyük resmi anlamak önemlidir; genel mimari ve yapıyı iyi özümsemek gerekir.

Projemiz bir graf (ağ) yapısı üzerine kuruludur. Bina içindeki her koridor kavşağı bir düğüm (node), iki düğümü birleştiren her koridor ise bir kenar (edge) olarak modellenir. Bir koridor kapandığında sistem, BFS algoritmasıyla alternatif rotayı hesaplar ve tüm düğümleri kablosuz olarak günceller.

5 düğüm, 7 kenar — Her düğümde bir ESP32 + 8×8 Dot Matrix + koridora özel buton(lar)

🔲

Düğüm (Node) = ESP32 Cihazı

Her kavşak noktasına ESP32'li bir cihaz yerleştirilir. Bu yapıda ayrıca dot matrix ekran ve o düğümden çıkan koridorların sayısı kadar buton bulunur.

📡

Mesh Ağ = ESP-NOW

Düğümler internet veya router olmadan birbirleriyle kablosuz iletişim kurar. Bir butona basıldığında "koridor X kapandı" mesajı tüm ağa yayılır.

🧮

Rota = BFS Algoritması

Her ESP32, BFS algoritmasıyla mevcut açık koridorlardan en kısa yolu hesaplar. Sonucu dot matrix ekranda yön oku olarak gösterir.

🔋

Elektriksiz Çalışma

Her düğüm Li-ion pil + MT3608 step-up regülatör ile beslenir. MT3608 çıkışı 5V'a ayarlanır; hem ESP32 hem MAX7219 bu hattan beslenir. Elektrik kesilse bile sistem çalışmaya devam eder.

💡
Buton Tasarım Mantığı (Güncelleme) Her düğümde, o düğümden çıkan koridor sayısı kadar buton bulunur. Örneğin A düğümü B ve C koridorlarına bağlıysa A düğümündeki ESP32'ye 2 buton bağlıdır. Bu sayede hangi buton basıldıysa, tam olarak hangi koridorun kapandığı bilinir.

🎯 Koridorlara sensörler (duman, alev vb) monte edilerek sistemin tamamen otonom çalışması sağlanabilir.

8×8 Dot Matrix Ekran

MAX7219 sürücülü modüle bağlantı ve yön oku gösterimi

Projede OLED yerine 8×8 LED Dot Matrix (MAX7219) kullanıyoruz. Bu ekran, 64 LED'den oluşan bir matris üzerinde harf, rakam veya şekil gösterebilir. SPI protokolüyle haberleşir ve sadece 5 kablo gerektirir.

🔌 Bağlantı Şeması — MAX7219 → ESP32
VCC→5V — MT3608 çıkışı (ESP32 Vin ile ortak hat)
GND→GND
DIN (MOSI)→GPIO 23
CLK (SCK)→GPIO 18
CS (SS)→GPIO 5
⚡ Güç bağlantısı (ortak hat):
Li-ion → MT3608 (5V) → ESP32 Vin pini  ve  MAX7219 VCC
GND → ESP32 GND  ve  MAX7219 GND (ortak)
📦 Kütüphane Kurulumu
Arduino IDE'de:
Sketch → Include Library → Manage Libraries
→ "MD_MAX72XX" ara → yükle
→ "MD_Parola" ara → yükle

✓ Her iki kütüphane de birlikte kullanılır

⚡ Donanım notu:
MT3608 trimpotunu multimetreyle tam 5.0V'a ayarla. Sonra ESP32 Vin'e ve MAX7219 VCC'ye bağla.

Buton Mantığı

Her koridor için ayrı buton — hangisi basıldı, o kapanır

Projenin en kritik tasarım kararı: her düğümde, o düğümden çıkan koridor sayısı kadar buton var. Bu sayede sistem, hangi koridorun kapandığını otomatik olarak biliyor.

🔵

Düğüm A (BAŞLANGIÇ)

B ve C koridorlarına bağlı → 2 buton
Buton 1: A–B kapandı
Buton 2: A–C kapandı

🔵

Düğüm B

A, C ve D koridorlarına bağlı → 3 buton
Buton 1: B–A kapandı
Buton 2: B–C kapandı
Buton 3: B–D kapandı

🟢

Düğüm C

A, B, D ve E'ye bağlı → 4 buton
Buton 1: C–A kapandı
Buton 2: C–B kapandı
Buton 3: C–D kapandı
Buton 4: C–E kapandı

🟡

Düğüm D

B, C ve E koridorlarına bağlı → 3 buton
Buton 1: D–B kapandı
Buton 2: D–C kapandı
Buton 3: D–E kapandı

🔴

Düğüm E (ÇIKIŞ)

Çıkış düğümüne buton gerekmez — sistem buraya ulaşmaya çalışır.

Buton Devresi ve Kodu

ℹ️
Pull-Down Direnç Zorunlu! Butonu direkt GPIO'ya bağlarsan pin "havada" kalır ve yanlış okuma yapar. Her buton için 10kΩ direnç kullan veya kodda INPUT_PULLUP kullan.
buton_test.ino — Düğüm A örneği (2 buton)
// DÜĞÜM A — B koridoru butonu GPIO 12, C koridoru butonu GPIO 14
// INPUT_PULLUP: basılınca LOW okur, bırakılınca HIGH

const int BUTON_AB = 12;  // A–B koridoru butonu
const int BUTON_AC = 14;  // A–C koridoru butonu

bool ab_kapali = false;
bool ac_kapali = false;

void setup() {
  Serial.begin(115200);
  pinMode(BUTON_AB, INPUT_PULLUP);
  pinMode(BUTON_AC, INPUT_PULLUP);
}

void butonlariKontrolEt() {
  // Butona basılınca LOW gelir (PULLUP mantığı)
  if (digitalRead(BUTON_AB) == LOW) {
    delay(50);  // debounce — titreşimi önle
    if (digitalRead(BUTON_AB) == LOW) {
      ab_kapali = !ab_kapali;  // toggle: kapat/aç
      Serial.println(ab_kapali ? "A-B KAPALI" : "A-B ACIK");
      // → ESP-NOW ile diğer düğümlere bildir (Bkz. Bölüm 08)
      while(digitalRead(BUTON_AB) == LOW); // bırakılana kadar bekle
    }
  }

  if (digitalRead(BUTON_AC) == LOW) {
    delay(50);
    if (digitalRead(BUTON_AC) == LOW) {
      ac_kapali = !ac_kapali;
      Serial.println(ac_kapali ? "A-C KAPALI" : "A-C ACIK");
      while(digitalRead(BUTON_AC) == LOW);
    }
  }
}

void loop() {
  butonlariKontrolEt();
}
💡
Debounce Nedir? Fiziksel bir butona bastığında mekanik titreşim yüzünden birkaç milisaniye içinde ON-OFF-ON sinyali üretebilir. delay(50) ile beklersek bu titreşim geçer ve gerçek basışı okuruz. Buna "debounce" denir.

BFS Algoritması

En kısa güvenli rotayı bulmak — önce kağıtta, sonra kodda

BFS (Breadth-First Search / Genişlik Öncelikli Arama), bir grafta en kısa yolu bulan temel bir algoritmadır. "Dalgalar halinde yayılan" bir arama gibi düşün: başlangıç noktasından eş zamanlı olarak tüm komşu yönlere gider.

🎮 İnteraktif BFS Simülatörü — Koridorları kapat, rotanın nasıl değiştiğini gör

Kapatmak istediğin koridoru seç:

BFS hazır. Koridor kapat veya "BFS Hesapla" düğmesine bas...

Graf Komşuluk Matrisi

Graf yapısı kodda bir 2 boyutlu dizi ile temsil edilir. Satır ve sütun sırası: A=0, B=1, C=2, D=3, E=4

ABCDE
A01100
B10110
C11011
D01101
E00110

BFS Kodu — ESP32 Üzerinde

bfs_algoritma.ino
// Graf komşuluk matrisi (5 düğüm: A=0, B=1, C=2, D=3, E=4)
int graf[5][5] = {
  // A  B  C  D  E
  {  0, 1, 1, 0, 0 },  // A
  {  1, 0, 1, 1, 0 },  // B
  {  1, 1, 0, 1, 1 },  // C
  {  0, 1, 1, 0, 1 },  // D
  {  0, 0, 1, 1, 0 }   // E (çıkış)
};

bool kapali[5][5] = { false };  // Tüm koridorlar başta açık

// BFS ile en kısa yolu bul
// baslangic: hangi düğümde (0-4), hedef: çıkış düğümü (genellikle 4)
// rota[]: bulunan yol, rotaUzunluk: kaç düğüm var
bool bfs(int baslangic, int hedef, int rota[], int &rotaUzunluk) {
  bool ziyaret[5] = { false };
  int  onceki[5];
  memset(onceki, -1, sizeof(onceki));

  int kuyruk[20];
  int on = 0, arka = 0;
  kuyruk[arka++] = baslangic;
  ziyaret[baslangic] = true;

  while (on < arka) {
    int mevcut = kuyruk[on++];

    if (mevcut == hedef) {
      // Hedefe ulaşıldı — rotayı geri izle
      rotaUzunluk = 0;
      int tmp = hedef;
      while (tmp != -1) {
        rota[rotaUzunluk++] = tmp;
        tmp = onceki[tmp];
      }
      // Diziyi tersine çevir (E→A → A→E)
      for (int i = 0; i < rotaUzunluk/2; i++) {
        int t = rota[i];
        rota[i] = rota[rotaUzunluk-1-i];
        rota[rotaUzunluk-1-i] = t;
      }
      return true;
    }

    // Komşuları kuyruğa ekle
    for (int k = 0; k < 5; k++) {
      if (graf[mevcut][k]           // bağlantı var mı?
          && !kapali[mevcut][k]     // koridor açık mı?
          && !ziyaret[k]) {          // daha önce ziyaret edilmedi mi?
        ziyaret[k] = true;
        onceki[k] = mevcut;
        kuyruk[arka++] = k;
      }
    }
  }
  return false;  // Yol bulunamadı
}

// Düğümden çıkış yönünü belirle
int cikisYonuBul(int benimID) {
  int rota[10];
  int uzunluk;
  if (bfs(benimID, 4, rota, uzunluk) && uzunluk > 1) {
    return rota[1];  // Bir sonraki düğüm = gidilecek yön
  }
  return -1;  // Yol yok
}
💡
cikisYonuBul() nasıl çalışır? BFS tüm rotayı döndürür (örn: A→C→E). Ama dot matrix ekranda sadece "bir sonraki adım" gösterilecek. rota[1] her zaman bir sonraki düğümü verir. Bu bilgiyi dot matrix'te yön okuna dönüştürürsün.

ESP-NOW Mesh Ağı

İnternetsiz kablosuz haberleşme — afet anında bile çalışır

ESP-NOW, ESP32'lerin Wi-Fi router'a ihtiyaç duymadan birbirleriyle doğrudan konuşmasını sağlayan bir protokoldür. Bluetooth'a benzer ama çok daha hızlı ve uzun menzilli.

⚡

Hız

Mesaj iletimi genellikle 1–5 ms içinde gerçekleşir. 3 saniye içinde tüm ağ güncellenmesi hedefi kolayca karşılanır.

📶

Menzil

Açık alanda ~200 m, bina içinde ~30–50 m. Maket testleri için fazlasıyla yeterli.

🔑

MAC Adresi

Her ESP32'nin benzersiz bir MAC adresi vardır. Mesaj gönderilecek ESP32'nin MAC adresini önceden kaydetmen gerekir.

1

MAC Adreslerini Öğren

Her ESP32'ye sırayla bu kodu yükle ve Serial Monitor'da MAC adresini not et. Her karta etiket yap (D1, D2, D3...).

mac_adres.ino
#include <WiFi.h>
void setup() {
  Serial.begin(115200);
  Serial.print("MAC: ");
  Serial.println(WiFi.macAddress());
}
void loop() {}
2

Mesaj Yapısını Tanımla

Tüm ESP32'ler aynı struct yapısını kullanmalı. Bir düğümde buton basıldığında bu yapı diğerlerine gönderilir.

mesaj_yapisi.h
// Tüm düğümlerde aynı struct tanımlanmalı!
struct Mesaj {
  int gonderenID;   // Hangi düğüm gönderiyor? (0=A, 1=B...)
  int dugum1;       // Kapanan koridorun başlangıç düğümü
  int dugum2;       // Kapanan koridorun bitiş düğümü
  bool kapaliMi;   // true = kapalı, false = tekrar açıldı
};

  {0xAA, 0xBB, 0xCC, 0x77, 0x88, 0x99},  // Düğüm D
  {0xAA, 0xBB, 0xCC, 0xAA, 0xBB, 0xCC},  // Düğüm E
};

void gonderimSonucu(const uint8_t *mac, esp_now_send_status_t durum) {
  Serial.println(durum == ESP_NOW_SEND_SUCCESS ? "✓ Gönderildi" : "✗ Gönderilemedi");
}

void setup() {
  Serial.begin(115200);
  WiFi.mode(WIFI_STA);
  esp_now_init();
  esp_now_register_send_cb(gonderimSonucu);

  // Tüm alıcıları "peer" olarak ekle
  for (int i = 0; i < 4; i++) {
    esp_now_peer_info_t peer = {};
    memcpy(peer.peer_addr, macler[i], 6);
    peer.channel = 0;
    peer.encrypt = false;
    esp_now_add_peer(&peer);
  }
}

void mesajYayinla(int d1, int d2, bool kapali) {
  Mesaj m;
  m.gonderenID = 0;  // Bu ESP32 Düğüm A ise 0
  m.dugum1 = d1;
  m.dugum2 = d2;
  m.kapaliMi = kapali;

  // Tüm düğümlere gönder
  for (int i = 0; i < 4; i++) {
    esp_now_send(macler[i], (uint8_t *)&m, sizeof(m));
  }
}

void rotayiGuncelle() {
  int benimID = 1;  // Bu ESP32 hangi düğüm?

Tam Entegrasyon

Tüm parçaları bir araya getiren ana program akışı

Her ESP32'ye yüklenecek ana program şu adımları izler:

1

setup() — Başlangıç

MT3608 (5V) → ESP32 Vin + MAX7219 VCC beslemesi hazır → Dot matrix başlat → Butonları tanımla → ESP-NOW başlat → Diğer düğümleri peer olarak ekle → İlk BFS hesapla → Ekranı göster

2

loop() — Sürekli Döngü

Butonları kontrol et → Değişiklik varsa ESP-NOW ile yayınla + yerel graf matrisini güncelle + BFS yeniden hesapla + ekranı güncelle

3

mesajAlindi() callback — Dışarıdan gelen mesaj

Gelen struct'ı parse et → Graf matrisini güncelle → BFS yeniden hesapla → Ekranı güncelle

main_dugum.ino — Tam entegrasyon şablonu
// ===================================================
// AcilRota — Düğüm Ana Programı
// Her ESP32'ye bu kodu yükle, sadece BENIM_ID'yi değiştir
// ===================================================
#include <esp_now.h>
#include <WiFi.h>
#include <MD_MAX72xx.h>
#include <SPI.h>

#define BENIM_ID  0   // 0=A, 1=B, 2=C, 3=D, 4=E
#define HEDEF_ID  4   // E = çıkış düğümü
#define N         5   // Toplam düğüm sayısı

// Pin tanımları
#define DATA_PIN 23
#define CLK_PIN  18
#define CS_PIN   5

// Düğüm A için buton pinleri (her düğümde değişir!)
// A düğümü: B ve C koridorlarına bağlı → 2 buton
struct ButonKoridor { int pin; int d1; int d2; };
ButonKoridor butonlar[] = {
  { 12, 0, 1 },  // GPIO12 → A-B koridoru
  { 14, 0, 2 },  // GPIO14 → A-C koridoru
};
const int BUTON_SAYISI = 2;

// Graf ve kapalı dizileri (Bölüm 06'dan)
int  graf[N][N] = { ... };
bool kapali[N][N] = { false };

MD_MAX72XX mx(MD_MAX72XX::FC16_HW, DATA_PIN, CLK_PIN, CS_PIN, 1);

void setup() {
  mx.begin();
  mx.control(MD_MAX72XX::INTENSITY, 5);
  for (int i=0; i<BUTON_SAYISI; i++) pinMode(butonlar[i].pin, INPUT_PULLUP);
  WiFi.mode(WIFI_STA);
  esp_now_init();
  esp_now_register_recv_cb(mesajAlindi);
  peerleriEkle();  // Diğer ESP32'leri kaydet
  rotayiGuncelle();
}

void loop() {
  for (int i=0; i<BUTON_SAYISI; i++) {
    if (digitalRead(butonlar[i].pin) == LOW) {
      delay(50);
      if (digitalRead(butonlar[i].pin) == LOW) {
        bool yeniDurum = !kapali[butonlar[i].d1][butonlar[i].d2];
        kapali[butonlar[i].d1][butonlar[i].d2] = yeniDurum;
        kapali[butonlar[i].d2][butonlar[i].d1] = yeniDurum;
        mesajYayinla(butonlar[i].d1, butonlar[i].d2, yeniDurum);
        rotayiGuncelle();
        while(digitalRead(butonlar[i].pin) == LOW);
      }
    }
  }
}

Sorun Giderme

Takıldığında buraya bak

ESP32 bilgisayarda görünmüyor / Port seçilemiyor ▾
CP2102 veya CH340 USB-Seri sürücüsünü yükle. ESP32 kartın üzerindeki küçük chip'in adına bak. Google'da "[chip adı] driver download" şeklinde ara. Sürücüyü yükledikten sonra Arduino IDE'yi kapat-aç ve Tools → Port'u kontrol et. Farklı bir USB kablosu da dene (bazı kablolar sadece şarj yapar, data iletmez).
Upload başarısız — "Connecting..." takılı kalıyor ▾
Upload başlarken Serial Monitor'da "Connecting..." yazısı görünür görünmez ESP32 üzerindeki BOOT butonuna basılı tut. Upload işlemi başlayınca (ilerleme çubuğu çıkınca) bırak. Bazı kartlarda BOOT ve EN butonlarına birlikte basmak gerekir.
Dot Matrix ekran yanmıyor / karanlık kalıyor ▾
1) Bağlantıları kontrol et: VCC→5V (Vin), GND→GND, DIN→23, CLK→18, CS→5. 2) mx.begin() çağrıldığından emin ol. 3) mx.control(MD_MAX72XX::INTENSITY, 8) ile parlaklığı artır. 4) mx.clear()'dan sonra mx.update() veya setRow() çağırdığından emin ol. 5) MD_MAX72XX kütüphanesi kuruluyken donanım tipini FC16_HW seç.
BFS yanlış rota gösteriyor ▾
Graf matrisini kağıda çiz ve elle takip et. Serial Monitor'a kapali[i][j] değerlerini yazdır — hangi kenarlar kapalı görünüyor? Unutma: koridor A–B kapatıldığında hem kapali[0][1] hem kapali[1][0] true olmalı (yönlü değil, yönsüz graf). Ayrıca BFS'in başlangıç ve hedef parametrelerini kontrol et.
ESP-NOW mesajı gitmiyor ▾
1) MAC adreslerini doğrula: her ESP32'ye WiFi.macAddress() kodunu yükle ve Serial Monitor'a bak. 2) esp_now_add_peer() başarı dönüp dönmediğini kontrol et. 3) Alıcı ESP32 da esp_now_init() çağırmalı. 4) Her iki cihaz da WiFi.mode(WIFI_STA) modunda olmalı. 5) Gönderici callback'te başarı mesajı var mı bak: ESP_NOW_SEND_SUCCESS
Buton çok fazla tetikleniyor (titreşim sorunu) ▾
Debounce eksik. Buton okumasından sonra delay(50) ekle ve bırakıldığına emin ol: while(digitalRead(pin) == LOW); satırı butona basılı tutulduğunda sistemi kilitler. Bu satırı da ekle. Alternatif olarak millis() tabanlı non-blocking debounce kullanabilirsin.
Pil hızla bitiyor ▾
ESP32 aktif modda ~240mA çeker. Dot matrix de enerji harcar. Önce: loop() içindeki delay() değerlerini artır (100ms yeterli). İleri: ESP-NOW mesajı bekliyorken light sleep moduna al. Dot matrix parlaklığını düşür: mx.control(INTENSITY, 2).
